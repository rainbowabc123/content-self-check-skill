# 示例 4：排版规范和专业术语

## 原始文案

```
我们使用AI技术，让ROI提升了50%！！！效果真的太好了。

通过Machine Learning算法和Deep Learning模型，我们的KPI增长显著。这个DL模型的准确率达到95%，而这个深度学习系统的准确率是96%。

关于Python编程，大家可以参考官方文档。使用pandas进行数据分析，用numpy处理数值计算。

我们的产品适合所有人使用，无论你是technical还是non-technical background。
```

## 检测结果

### 💡 建议优化

**1. [排版规范] 中英文之间缺少空格**

> 使用AI技术... ROI提升了50%

**问题说明**：遵循《中文文案排版指北》，中文与英文、中文与数字之间应该有空格。

**建议修改**：
- `使用AI技术` → `使用 AI 技术`
- `ROI提升了50%` → `ROI 提升了 50%`

---

**2. [排版规范] 重复标点**

> 效果真的太好了。。。

**问题说明**：不应使用重复标点符号（`！！！`、`。。。`），应使用单个标点。

**建议修改**：`太好了。。。` → `太好了！` 或 `太好了。`

---

### ⚠️ 中等问题

**3. [专业术语] 术语不统一**

> Deep Learning 模型... 这个 DL 模型... 这个深度学习系统

**问题说明**：同一概念使用了三种不同表述（Deep Learning、DL、深度学习），容易造成混淆。

**建议修改**：统一使用一种表述，建议使用"深度学习"或"Deep Learning"，首次出现时可以注明"深度学习（Deep Learning，简称 DL）"

---

**4. [专业术语] 术语未解释**

> 我们的 KPI 增长显著

**问题说明**：如果目标读者不是专业人士，"KPI"可能需要解释。

**建议修改**：首次使用时解释："关键绩效指标（KPI）增长显著"

---

**5. [排版规范] 代码和工具名称格式不统一**

> Python... pandas... numpy

**问题说明**：工具名称的大小写不统一。

**建议修改**：
- `pandas` → `Pandas`（或保持小写但统一）
- `numpy` → `NumPy`（官方写法）

---

**6. [中英文混用] 不必要的英文**

> technical 还是 non-technical background

**问题说明**：可以用中文表达的概念不必使用英文，除非是专业术语。

**建议修改**：`technical 还是 non-technical background` → `技术背景还是非技术背景`

---

## 修改后的文案

```
我们使用 AI 技术，让 ROI 提升了 50%！效果真的很好。

通过机器学习算法和深度学习（Deep Learning，简称 DL）模型，我们的关键绩效指标（KPI）增长显著。这个深度学习模型的准确率达到 95%，优化版本的准确率达到 96%。

关于 Python 编程，大家可以参考官方文档。使用 Pandas 进行数据分析，用 NumPy 处理数值计算。

我们的产品适合所有人使用，无论你是技术背景还是非技术背景。
```

## 中文排版规范速查

根据《中文文案排版指北》：

| 规则 | 错误示例 | 正确示例 |
|------|---------|---------|
| 中英文空格 | `使用AI技术` | `使用 AI 技术` |
| 中文数字空格 | `增长50%` | `增长 50%` |
| 全角标点 | `你好.` | `你好。` |
| 半角数字 | `增长５０%` | `增长 50%` |
| 不重复标点 | `真好！！！` | `真好！` |
| 链接空格 | `访问GitHub网站` | `访问 GitHub 网站` |

---

## 专业术语使用建议

1. **首次出现时解释**：`关键绩效指标（KPI）`
2. **保持一致性**：全文统一使用同一术语
3. **考虑读者背景**：面向大众时少用行话
4. **中英文平衡**：能用中文就用中文，必要时标注英文
5. **缩写规范**：首次使用全称+缩写，之后可用缩写
